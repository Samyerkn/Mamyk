# Фронтенд C — Заявки и Спонсорство

Полная интеграция благотворительного пути: подача заявок нуждающимися и помощь от спонсоров.

## 📋 Реализованные страницы

### 1. **Публичная лента заявок** (`/requests`)
- **Компонент**: `HelpRequestFeed.jsx`
- **API**: `GET /api/requests/`
- **Функции**:
  - Показывает все активные заявки (статус `pending` или `in_progress`)
  - Карточки с прогресс-баром, отображающим процент сбора
  - Доступна без авторизации
  - Кнопка "Подать заявку" для авторизованных пользователей

### 2. **Форма подачи заявки** (`/requests/new`)
- **Компонент**: `HelpRequestForm.jsx`
- **API**: `POST /api/requests/`
- **Поля**:
  - Имя ребёнка
  - Диагноз
  - История (story) — текстовое поле
  - Тип товара (buttons, magnets, velcro)
  - Размер футболки (S, M, L и т.д.)
  - Необходимая сумма в ₸
- **Маршрутизация**: Может быть открыта с предзаполненными `type` и `size` из Фронтенда B

### 3. **Детальная страница заявки** (`/requests/:id`)
- **Компонент**: `RequestDetail.jsx`
- **API**: 
  - `GET /api/requests/{id}/` — получить данные заявки
  - `POST /api/donations/` — создать пожертвование
  - `GET /api/donations/by_request/{id}/` — история пожертвований
- **Функции**:
  - История ребёнка и диагноз
  - Прогресс-бар с live-обновлением после взноса
  - Форма для ввода суммы пожертвования
  - Кнопка "Закрыть полностью" — автоматически рассчитывает оставшуюся сумму
  - История всех пожертвований на эту заявку
  - При достижении 100% появляется уведомление о завершении

### 4. **Мои заявки** (`/requests/me`)
- **Компонент**: `MyHelpRequests.jsx`
- **API**: `GET /api/requests/me/`
- **Функции**:
  - Показывает все заявки текущего пользователя
  - Статус каждой заявки и процент сбора
  - Требует авторизации

### 5. **История помощи спонсора** (`/donations/history`)
- **Компонент**: `SponsorHistory.jsx`
- **API**: `GET /api/donations/me/`
- **Функции**:
  - Показывает все пожертвования спонсора
  - Сумма, дата, статус полной оплаты
  - Общая сумма пожертвований
  - Требует авторизации

## 🔗 Интеграция с API (Бэкенд A)

### Авторизация
- Используется JWT токен из `/api/auth/register/` и `/api/auth/login/`
- Токен сохраняется в `localStorage` как `access_token`
- Передаётся в заголовке: `Authorization: Bearer {token}`

### Сервис `requests.js`
Все функции для работы с API:
- `fetchPublicHelpRequests()` — публичная лента
- `fetchHelpRequest(id)` — детали заявки
- `fetchMyHelpRequests()` — мои заявки (требует токена)
- `createHelpRequest(payload)` — создать заявку (требует токена)
- `createDonation(payload)` — создать пожертвование (требует токена спонсора)
- `fetchMyDonations()` — моя история (требует токена)
- `fetchDonationsByRequest(id)` — история по заявке (публичная)

## 🎯 Критерии готовности (Definition of Done)

✅ **24. Заявка создаётся через форму и сразу видна в публичной ленте**
- Форма сохраняет заявку через `POST /api/requests/`
- После сохранения заявка с статусом `pending` появляется в `GET /api/requests/`

✅ **25. Прогресс-бар корректно отражает amount_collected, обновляется после взноса без перезагрузки**
- Прогресс-бар вычисляется как `(amount_collected / amount_needed) * 100`
- После пожертвования страница вызывает `GET /api/requests/{id}/` и обновляет состояние React
- Бэкенд автоматически вычисляет `amount_collected` как сумму всех пожертвований

✅ **26. При полной оплате статус автоматически меняется на completed, завершённая заявка перестаёт отображаться в публичной ленте**
- Бэкенд при сохранении пожертвования вызывает `check_completion()` в модели `HelpRequest`
- Если `amount_collected >= amount_needed`, статус меняется на `COMPLETED`
- `GET /api/requests/` по умолчанию возвращает только `status__in=['pending', 'in_progress']`
- Завершённая заявка остаётся доступна по `GET /api/requests/{id}/` — архив

✅ **27. История помощи в кабинете спонсора показывает все его взносы**
- `GET /api/donations/me/` возвращает все пожертвования текущего пользователя
- Каждое пожертвование содержит сумму, дату, флаг полной оплаты
- История отсортирована по дате (самые свежие первыми)

## 🔄 Интеграция с Фронтендом B

При неудачной оплате на Фронтенде B может быть передача на `/requests/new` с параметрами:
- `?type=buttons` или `?type=magnets` или `?type=velcro`
- `?size=S` или `?size=M` и т.д.

Форма `HelpRequestForm.jsx` автоматически предзаполняет эти поля через `useSearchParams()`.
